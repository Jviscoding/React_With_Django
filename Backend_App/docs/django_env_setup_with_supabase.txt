============================================================

------------------------------------------------------------
WHY USE A .env FILE?
------------------------------------------------------------

A .env file stores configuration values separately from the source code.

Examples:
- Database credentials
- API keys
- Secret keys
- Email credentials

Benefits:
вњ” Keeps sensitive information out of source code
вњ” Easier to switch between development and production
вњ” Prevents accidentally committing secrets to Git
вњ” Allows multiple environments to use different configurations

------------------------------------------------------------
1. INSTALL PYTHON-DOTENV
------------------------------------------------------------

Install inside the project's virtual environment:

python -m pip install python-dotenv

Verify installation:

python -m pip show python-dotenv

------------------------------------------------------------
2. CREATE .env FILE
------------------------------------------------------------

Place the .env file in the project root
(the same directory as manage.py).

Example structure:

Project/

------------------------------------------------------------
3. ADD VARIABLES
------------------------------------------------------------

Example:

DB_NAME=postgres
DB_USER=postgres
DB_PASSWORD=my_database_password
DB_HOST=db.xxxxxxxxx.supabase.co
DB_PORT=5432

SECRET_KEY=your-secret-key
DEBUG=True

------------------------------------------------------------
4. LOAD THE .env FILE
------------------------------------------------------------

At the top of settings.py:

import os
from dotenv import load_dotenv

load_dotenv()

This loads all variables from the .env file into Python's
environment variables.

------------------------------------------------------------
5. ACCESS VARIABLES
------------------------------------------------------------

Retrieve values using:

os.getenv("VARIABLE_NAME")

Example:

DATABASES = {
    "default": {
        "ENGINE": "django.db.backends.postgresql",
        "NAME": os.getenv("DB_NAME"),
        "USER": os.getenv("DB_USER"),
        "PASSWORD": os.getenv("DB_PASSWORD"),
        "HOST": os.getenv("DB_HOST"),
        "PORT": os.getenv("DB_PORT"),
        "OPTIONS": {
            "sslmode": "require",
        },
    }
}

------------------------------------------------------------
6. VERIFY VARIABLES
------------------------------------------------------------

Temporary debugging:

print(os.getenv("DB_NAME"))
print(os.getenv("DB_HOST"))

If they print None:

- .env not found
- load_dotenv() missing
- Variable name misspelled
- .env in wrong directory

------------------------------------------------------------
7. TEST DATABASE CONNECTION
------------------------------------------------------------

Run:

python manage.py migrate

If successful:

вњ” Django connects to the database.
вњ” Migration tables are created.
вњ” Future models can be migrated normally.
