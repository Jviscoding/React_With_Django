
------------------------------------------------------------
CORE IDEA
------------------------------------------------------------

Django can be extended using third-party packages.

These packages:
→ Are built by the community  
→ Add features without writing everything from scratch  
→ Plug directly into Django projects  

Example:
→ Django Debug Toolbar (https://django-debug-toolbar.readthedocs.io/en/latest/installation.html)
→ Read the link for installation instructions 

------------------------------------------------------------
1. THIRD-PARTY PACKAGES
------------------------------------------------------------

Meaning:
External libraries that integrate with Django

Why they are useful:
→ Save development time  
→ Add powerful features quickly  
→ Common in real-world projects  

Examples:
- Django Debug Toolbar
- Django REST Framework
- django-allauth

------------------------------------------------------------
2. INSTALLING A PACKAGE
------------------------------------------------------------

Command:
  python -m pip install django-debug-toolbar

Meaning:
→ Installs the package into your virtual environment

IMPORTANT:
→ Always activate your virtual environment first

------------------------------------------------------------
3. INTEGRATING INTO DJANGO
------------------------------------------------------------

After installing a package, you usually need setup steps:

Common steps:
1. Add to INSTALLED_APPS
2. Update urls.py (if required)
3. Add middleware (if needed)
4. Configure settings

------------------------------------------------------------
DJANGO DEBUG TOOLBAR
------------------------------------------------------------

What it is:
→ A debugging tool for Django development

What it shows:
- SQL queries
- Request/response details
- Cache information
- Template rendering info
- Performance insights

------------------------------------------------------------
HOW TO USE IT
------------------------------------------------------------

1. Run server:
   python manage.py runserver

2. Open:
   http://localhost:8000/admin/

3. Look for toolbar on the right side

4. Click it to expand panels

------------------------------------------------------------
4. TROUBLESHOOTING PACKAGES
------------------------------------------------------------

If something doesn’t work:

Check:
- Official documentation
- GitHub issue tracker
- Package FAQs

Ask for help:
- Django Forum
- Django Discord

------------------------------------------------------------
5. CHOOSING GOOD PACKAGES
------------------------------------------------------------

Good packages usually:
→ Actively maintained  
→ Recently updated  
→ Well documented  
→ Widely used  

Red flags:
→ Outdated  
→ No recent updates  
→ Unresolved issues  

------------------------------------------------------------
KEY TAKEAWAY
------------------------------------------------------------

- Django supports third-party packages
- pip installs them into your environment
- You must configure them manually in settings
- Debug Toolbar helps inspect requests and SQL
- Always check package quality before using
